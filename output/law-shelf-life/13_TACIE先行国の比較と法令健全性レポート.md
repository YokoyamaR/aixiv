# 第13〜14ラウンド：TACIE先行国の比較研究と法令健全性レポートのプロトタイプ

## TACIE先行事例の詳細分析

### 事例1：台湾（vTaiwan / Join）

| 項目 | 内容 |
|------|------|
| 開始 | 2014年（vTaiwan）、2017年（Join） |
| 主導者 | オードリー・タン（唐鳳） |
| 技術基盤 | Pol.is（AI合意形成ツール） |
| 規模 | 数千人規模の市民参加 |
| 成果 | ライドシェア規制、遠距離酒類販売規制、FinTech規制法案の立法化 |
| Join実績 | 5,000件以上の市民提案 |

**成功要因：**
1. 政治的リーダーシップ（タンの存在）
2. 小規模で均質な社会
3. 高いデジタルリテラシー

**限界：**
- 参加者バイアス（テック層に偏る）
- 政治的に敏感なテーマ（中台関係）は回避
- 大国への直接適用は困難

### 事例2：エストニア（デジタル国家）

| 項目 | 内容 |
|------|------|
| 人口 | 約130万人 |
| 特徴 | 行政の99%がオンライン化 |
| 技術基盤 | X-Road（分散型データ基盤） |
| TACIEとの関連 | 法改正時の既存システムへの自動影響チェック |

- 法案の影響評価がデジタル化 = **TACIEフェーズ1の原型**
- AI裁判官は計画段階で議論停滞
- e-Residency（電子市民権）は法域選択の萌芽

### 事例3：シンガポール（規制サンドボックス）

| 項目 | 内容 |
|------|------|
| 領域 | FinTech中心 |
| 方法 | 暫定的規制枠組みでテスト→正式規制策定 |
| 原則 | FEAT（Fairness, Ethics, Accountability, Transparency） |
| 意義 | 「法の常時更新」の一形態 |

## 先行事例の共通パターン（中村の分析）

| パターン | 説明 | 日本との比較 |
|---------|------|------------|
| 小国 | 人口少、意思決定速い | 日本は1.2億人、合意形成が遅い |
| テック指向リーダー | 政治的リーダーシップ | 日本にはタン的人物が不在 |
| 段階的導入 | 特定領域から開始 | 日本は「全体を変えるか何も変えないか」になりがち |
| 外的動機（生存圧力） | 変革しなければ生き残れない | 日本は「変わらなくてもなんとかなる」（今のところ） |

**李の補足：** 日本に外的動機が生まれるとすれば、人口減少と経済的相対化が進んだ時。

**山本の補足：** 台湾から学ぶべきは「粗い合意」の文化。日本の「全会一致」志向がTACIE導入の文化的障壁。

## 法令健全性レポート・プロトタイプ設計（サラ）

### システム構成

```
[入力データ]
├── e-Gov全法令データベース
├── 国会議事録
├── 最高裁判例データベース
├── 各省庁の通達・ガイドライン
└── 主要学術論文

    ↓ AI処理

[分析エンジン]
├── 法令間依存関係グラフ構築
├── 解釈負債スコア算出
├── 制定時前提条件の妥当性評価
└── 国際比較ベンチマーク

    ↓

[出力]
├── 法令健全性ダッシュボード（全法令の健康状態を可視化）
├── 解釈負債ランキング（リスク順位）
├── 改正優先度レポート
└── トレンド分析（負債蓄積速度の加速度）
```

### 哲学的制約（田中・鈴木）

**ヒュームの法則の適用：**
- AIが言えること：「この法律は制定時から大きく乖離している」（事実判断）
- AIが言えないこと：「だから変えるべきだ」（価値判断）
- 古い法律にも守る価値があるかもしれない
- **「である」から「べきである」は導出できない**
- AIは「事実としての状態」を提示し、「あるべき姿」の判断は人間に委ねる
