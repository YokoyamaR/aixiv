# 第5ラウンド：具体的な適用領域と、「AI時代の民主主義」の変容

## 日本における「耐用年数切れ」法律の具体例

### 1. 労働基準法（1947年制定）
- 工場労働者を前提とした設計
- リモートワーク・ギグワーカー・副業に非対応
- 「1日8時間、週40時間」の画一的規制の限界
- 2018年働き方改革は部分的手当てに過ぎない

### 2. 戸籍法（原型1871年、現行法1947年）
- 「家」単位の思想が残存
- 選択的夫婦別姓・同性パートナーシップ・事実婚に非対応
- 国際結婚の増加にも追いついていない

### 3. 著作権法
- 生成AIにより「著作物」「創作性」「利用」の概念が根底から動揺
- 30条の4（機械学習のための情報解析の許容）は現在の状況を想定していない
- 世界的に訴訟が激増、立法が追いつかない典型

### 4. 会社法（2005年制定）
- DAO等の新組織形態に非対応
- ドイツ・イギリスと比較して構造的見直しが少ない

### 5. 税法
- 給与所得者中心の設計
- 暗号資産・NFT・クロスボーダーデジタル取引に対応しきれない
- 定量的にシミュレーション可能 → **AIが最も威力を発揮する領域**

## TACIE試験導入の戦略（合意形成）

### 原則：政治的にセンシティブでない領域から始める（山本・デュポン）
- 国論を二分するテーマ（憲法9条、家族法）からの着手は危険
- TACIEシステム自体が政治的対立に巻き込まれて潰される
- ナポレオン法典の戦略：日常生活の法から着手し、市民の実利を示す

### 推奨される試験導入の優先順位

| 優先度 | 法律領域 | 理由 |
|--------|----------|------|
| 高 | 行政手続法・情報公開法 | 地味だが改善余地大。政治的対立が少ない |
| 高 | 税法（技術的側面） | 定量的シミュレーションが可能 |
| 中 | 労働基準法 | 社会的ニーズ大だが利害関係者が多い |
| 中 | 著作権法 | 緊急性高いが国際的調整が必要 |
| 低（当面回避） | 憲法 | 国論を二分するリスク |
| 低（当面回避） | 家族法 | 価値観の対立が深い |

## 民主主義モデルの転換

### バッチ処理型 → ストリーミング型（鈴木）

| 特性 | バッチ処理型（現行） | ストリーミング型（TACIE） |
|------|---------------------|------------------------|
| 頻度 | 4年に一度の選挙 | 常時の政策意見表明 |
| 単位 | 代表者の選択（パッケージ） | 個別政策への意見（アンバンドル） |
| 規模 | 投票所での投票 | オンラインで1億人規模 |
| 先例 | 近代議会制民主主義 | 古代アテネの直接民主制（規模拡大版） |
| 実現条件 | 選挙インフラ | **AIなしには不可能** |

### 法のバージョン管理（藤田・サラ）
- 法律にバージョン番号、変更履歴の追跡
- フォーク（分岐）とマージ（統合）
- **GitLawプロジェクト**（ドイツ・フランスで実験中）
- ソフトウェア工学の知見を法制度に応用

## 安定性への配慮

### 法的安定性は機能であり欠点ではない（ロッシ）
- 予測可能性は自由の前提条件
- 「明日どんな法が適用されるかわからない」状態は恐怖
- **変化の速度に上限を設ける仕組みが不可欠**

### ラートブルフの公式による位置づけ（田中）
- 「法が不正に耐えられないほど矛盾する場合、正義が法に優越する」
- TACIEは限界点に達する**前に**問題を検出する**予防的メカニズム**
- 法的安定性と具体的妥当性の緊張を管理するツール

## 国際関係への影響（李）
- ストリーミング型とバッチ型の国家が並存
- ストリーミング型：機動的だが不安定
- バッチ型：安定だが硬直的
- 外交交渉における不確実性の増大
