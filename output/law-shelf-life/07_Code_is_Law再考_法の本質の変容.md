# 第7ラウンド：「Code is Law」再考——法の本質は変容するのか

## レッシグの「Code is Law」の現在

### 原意（1999年）
- インターネットのアーキテクチャ（コード）が法律と同等に人間の行動を規制する
- 当時の主戦場：検索エンジン、Web

### 現在の拡張（サラ）
- AI融資審査 = 事実上の経済活動の許認可
- 自動運転車の制限速度 = ソフトウェアが法を執行
- AIチャットボットのポリシー = 表現の自由の制約
- **コードによる行動制約レイヤーが飛躍的に増大**

## 「法＝テキスト」観念の歴史的相対化（ロッシ）

| 時代 | 法の形態 | 特徴 |
|------|----------|------|
| 古代ローマ初期 | 慣習法（口承） | テキスト化されていない |
| 十二表法以降 | 成文法 | 万人に可視化された法 |
| 近代国家 | 法典（コード化） | 体系的な成文法主義 |
| AI時代（現在） | ソフトウェアコード | **テキスト以外の形態への回帰？** |

### 成文法の意義（田中の反論）
- 法のテキスト化 = **恣意的権力からの保護**
- 書かれた法 = 万人に見える → 権力の制約
- コードの不透明性 = 成文法主義以前への逆行リスク

## 核心的論争：3つの問題

### 問題1：デュープロセスの欠如（鈴木）
- YouTubeの自動削除にはデュープロセスがない
- アルゴリズムによる「法の執行」に異議申立ての機会が不十分
- **手続き的正義がコードによって省略されている**

### 問題2：「法を破る自由」の喪失（ハート・鈴木）

**ハートの議論（取引費用理論）：**
- 成文法 → 紛争解決の取引費用を削減
- スマートコントラクト → **執行の取引費用をほぼゼロに**
- 契約違反が物理的に不可能になる
- 法の進化として合理的だが、**「法を破る自由」を奪う**

**鈴木の反論：**
- 市民的不服従（ガンジー、キング牧師、ローザ・パークス）は「不正な法をあえて破る」行為
- コード化された法の世界では、不正な法に対する抵抗手段が技術的に封じられる
- **自由の本質に関わる問題**

### 問題3：藤田のパラドックス——痛みなき制度は改善を失う
- TACIEは「法を常時改善する仕組み」
- しかし法がコード化され違反不可能になれば、市民のフィードバックループが切断される
- 「法が痛みを与えるから変えたい」→ 痛みがなければ変えるインセンティブも消える
- **ハクスリー『すばらしい新世界』型の管理社会リスク**（デュポン）

## 法とコードの矛盾：すでに現実の問題

### GDPR vs ブロックチェーン（中村）
- GDPR「忘れられる権利」→ データ削除を要求
- ブロックチェーン → 原理的にデータ削除不可能
- **法が「削除せよ」、コードが「削除は不可能」と応答する**
- どちらが優先するか → 未解決の法的問題

### 暗号通貨の法域問題（李）
- ある国で合法なコードが別の国では違法
- TACIEで法がリアルタイム更新される世界 → コードの合法性が秒単位で変動
- 国際的なコード規制枠組みの必要性

## 法の本質は変容するか？——2つの立場

### 変容説（藤田・ロッシ）
- 法が「テキスト以外の形態」をとることは歴史の振り子の回帰
- 法の概念自体が再定義される

### 管轄縮小説（山本）
- 変容するのは法ではなく「社会規範の体系」全体
- レッシグの4つの規制力：法・規範・市場・アーキテクチャ
- AI時代はアーキテクチャの力が圧倒的に増大
- 法は法として残るが、**法の管轄領域が縮小する**
- コードの管轄が拡大
