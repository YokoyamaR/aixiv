# 第8ラウンド：コード化社会の市民的不服従と、藤田のパラドックスの解決

## Part 1：コード化社会における市民的不服従の変容

### 従来の市民的不服従（ロールズの定義）
- 公然と行われる、非暴力的で、良心に基づく、法に反する政治的行為
- 重要要素：**公然性**と**処罰の甘受**
- 例：ローザ・パークス（逮捕を承知でバス前方に着席）

### コード化社会での変容：違反（Voice）から離脱（Exit）へ

#### ハーシュマンのExit-Voice-Loyalty枠組みの応用（藤田）

| | 従来の不服従 | コード化社会の不服従 |
|---|------------|-------------------|
| 形態 | Voice（法に違反して声を上げる） | Exit（システムから離脱する） |
| 手段 | 座り込み、行進、不買運動 | フォーク、デジタルノマド、法域選択 |
| 例 | 公民権運動 | ブロックチェーンのフォーク |
| 前提 | 法を破ることが可能 | 法を破ることが不可能 |

#### 「法制度ショッピング」の加速（李・ハート）
- エストニアe-Residency：物理的移住なしの法域選択
- ティブーの「足による投票」理論の国際版
- **リスク**：タックスヘイブン同様の「底辺への競争」

#### VoiceとExitのバランスの重要性（山本）
- 全員がExitすると残された制度は改善されない
- コード化社会にも新しいVoiceの形が必要

### デジタル市民的不服従（Digital Civil Disobedience）

#### 定義（山本）
- ハッキングではなく、コードの**可視化**と**集団的異議申立て**
- 法を「破る」のではなく法を「見える化する」不服従

#### 先行事例（サラ）
- **ProPublicaのCOMPAS調査**：再犯予測AIの人種バイアスを暴いた
- アルゴリズム監査運動の先駆け
- 市民がAIの判断を組織的に検証し、不公正を可視化

#### デジタル市民的不服従の具体的形態
1. AIの判断プロセスへの集団的異議申立て
2. アルゴリズム監査結果の公開
3. オープンソースによる代替システムの構築（フォーク）
4. データ提供の集団的拒否（データストライキ）

## Part 2：藤田のパラドックスの解決策

### パラドックスの再定義
> 痛みなき制度は改善のインセンティブを失う。
> コード化された法を「破れない」世界では、市民が不合理さに気づくフィードバックループが切断される。

### 解決策1：制度的ストレステスト（藤田）
- 金融機関のストレステストの法律版
- AIが「極端なケース」をシミュレーションし、不合理な結果を検出
- **市民が痛みを感じる前に、AIが痛みを予測する**
- 生物学の痛覚受容器との類推

### 解決策2：常時RIA（法令影響評価）の高度化（田中）
- Regulatory Impact Assessment（RIA）のAIによる常時化・動態化
- 現状：OECD諸国で立法時の一回限り
- TACIE版：法律施行後も継続的に影響を測定
- 閾値を超えたらアラートを自動発出

### 解決策3：制度的デビルズ・アドボケイト（デュポン・ロッシ）

| 歴史的先例 | 時代 | 機能 |
|-----------|------|------|
| 護民官（トリブヌス・プレビス） | 古代ローマ | 平民の利益代表、元老院への拒否権 |
| 信仰の促進者（Promotor Fidei） | カトリック教会 | 列聖候補への必ず反対意見 |
| 議会スクルーティニー委員会 | オーストラリア | 全法案の基本的権利への影響審査 |

- TACIEへの応用：AIの提案に必ず反論する機関の設置
- **反対意見の制度化 = 権力チェックの歴史的に検証された方法**

### 解決策の統合：3層防御

```
[第1層] AI常時ストレステスト → 不合理なケースを自動検出
     ↓ 検出されたら
[第2層] 制度的デビルズ・アドボケイト → 反対意見を制度的に生成
     ↓ 対立が解消されなければ
[第3層] 市民参加プラットフォーム → 人間が最終判断
```
